06-2 README 파일 작성하기
공개된 깃허브 저장소는 전 세계 누구나 구경할 수 있기 때문에 활동을 할수록 방문하는
사람이 늘어날 것이다. 하지만 프로필과 컨트리뷰션 그래프만 살펴보는 것으로는 그 
저장소의 주인이 어떤 활동을 했는지 구체적으로 알기 어렵다. 그래서 많은 깃허브 사용자는
방문자가 편안하게 자기 저장소를 살펴볼 수 있도록 안내문을 만들어 첫 화면에 
띄워 둔다. 이 문서를 README 파일이라고 한다. 지금까자 파일을 커밋하는 방법만
알아봤기 때문에 README.md 부분은 비워져 있을 것이다.

README 파일 만들기
    README 파일을 작성하기 전에 다른 저장소에서는 README 파일을 어떻게 사용하는지 
    살펴보자. README 파일은 마크다운(markdown) 문법을 사용하기 때문에 확장자는
    .md를 사용한다. 
    1. 먼저 README 파일을 만들 깃허브의 저장소로 접속 후 해당 Repository 선택
       해당 파일 목록 아래에 있는 [ADD a README] 클릭
    2. 텍스트 편집기에 원하는 내용을 입력한다. 이때 줄을 바꾸려면 Enter를 두 번 
       눌러서 가운데 빈 줄을 하나 넣어야 한다. 그래야 실제 화면에서 줄이 바뀌어
       나타난다.
    3. README 파일 작성 화면에는 [Edit new file]과 [Preview]가 있다. 
       [Edit new file]에서 작성한 후 [Preview]를 눌러 보면 결과 화면을 볼 수 있다.  
    4. 작성한 README 파일을 커밋하려면 먼저 화면 아래쪽에서 커밋 메시지를 작성
       기본으로 들어 있는 커밋 메시지를 그대로 사용할 수 있고 직접 입력할 수도 있다.
       작성을 끝내고 [Commit new file]을 눌러 커밋한다.
    5. 저장소 첫 화면으로 돌아와 파일 목록 아래에서 작성한 README 파일의 내용 확인
    6. 작성해 놓은 README 파일을 수정하려면 내용 오른쪽 연필 아이콘을 클릭
    7. README 파일을 수정한 후 [Preview changes]를 눌러 미리 보기 화면에서
       내용 왼쪽에 초록색 막대가 나타난 부분은 추가된 내용이고, 빨간색 막대가 나타난
       부분은 삭제된 내용이다.
    8. 커밋 메시지를 작성하고 [Commit changes]를 눌러 수정한 내용을 깃허브에 등록

마크다운 문법 알아보기
    제목 : 제목을 입력할 때는 텍스트 앞에 #을 붙인다. #(1개) ~ ######(6개) 제목글자크기
    텍스트 단락 줄 바꾸기 : Enter 두 번
    가로줄 : --- 또는 ---------- 또는 - - - 또는 *** 또는 ********** 또는 * * *
    순서목록 : 1,2,... 처럼 번호를 붙여서 
    순서업는목록 : +나 - 또는 *
    순서 없는 목록에서 Tab을 눌러 항목을 들여 쓰면 여러 단ㄱㅖ를 가진 목록이 만들어진다.
    테스트 강조 
        굵게 : 텍스트의 앞뒤를 **또는 __로 감싸
        기울임 : 테스트 앞뒤를 * 또는 _로 감싸
        굵은 기울임 : 텍스트 앞뒤를  *** 또는 ___로 감싸
        취소선 : 텍스트 앞뒤를 ~~로 감싸
    소스코드 : 텍스트 사이에 소스코드를 삽입하려면 ` 기호로 감싸
    여러줄 소스 삽입: ```javascript 
                        소스 
                    ```

    이미지
     마크다운에서 이미지를 삽입할 때는 맨 앞에 !를 붙인 후 대체 텍스트와 이미지 파일 경로를
     지정한다. 그런데 이미지 파일이 경로를 가지려면 웹에 올라오 있어야 한다. 웹 사이트를 
     운용하고 있다면 웹 사이트에 이미지 파일을 올린 뒤 그 경로를 가져와 사용

깃허브에 이미지 올리고 README에 삽입하기
    이미지 파일을 깃허브 저장소에 직접 올린 다음 그 경로를 README 파일에서 지정하면 된다.
    
    1. 먼저 사용자 컴퓨터에 이미지를 모아 둘 images 디렉터리를 만든다. 그리고 디렉터리 안에
       필요한 이미지들을 모아 두고 깃허브 저장소에 직접 이미지를 올릴 수도 있지만 커밋 관리
       중인 다른 파일들과 섞이지 않도록 별도의 디렉터리를 만들어 관리하는 것이 좋다.
       images 디렉터리는 어디에 두어도 상관없지만 계속해서 이미지를 추가해 사용할 예정이라면
       지역 저장소 안에 넣어 두는 것이 관리하기 쉽다.
    2. 이미지를 사용할 저장소에 접속한 후 [Add file] Upload files 를 누른다.
    3. 파일 탐색기에서 아까 만들어 둔 images 디렉터리를 누르고 깃허브 저장소로 끌어 올린다.
    4. 이미지 파일 올리기가 끝나면 [Commit changes]를 눌러 커밋한다.
    5. 깃허브 저장소에 images 디렉터리 생성 확인
    6. 올린 이미지 파일을 README 파일에 넣으려면 README.md 오른쪽에 있는 연필 아이콘 클릭
    7. 예를 들어 깃허브의 images 디렉터리에 있는 xxx.jpg 이미지를 삽입하려면
       ![...](./images/xxx.jpg) 입력
    8. README 수정이 끝나면 수정한 내용을 커밋
    9. README 안에 올리기 한 이미지 확인
