06-4 깃허브에 개인 블로그 만들기
    GitHub Pages 기능을 사용하면 별도의 웹 호스팅 서비스를 사용하지 않고도
    무료로 자기만의 홈페이지를 열 수 있다. 
    여기에서 이 기능을 사용해 개인 블로그를 만들어 보자

GitHub Pages 알아보기
    개인적으로 홈페이지를 열고 싶어서 사용자 컴퓨터에 홈페이지 파일을 만들어
    놓아도 웹 서버에 올리지 않으면 다른 사람에게 보여줄 수 없다. 그래서 웹
    서버 기능을 제공하는 사이트를 통해서 홈페이지를 연다. 이런 서비스를
    '웹 호스팅 서비스'라고 하는데, 대부분 매달 일정 금액을 지불해야 한다.
    하지만 깃허브에는 GitHub Pages라는 기능이 있다. 저장소에 홈페이지 파일을
    올린 뒤 저장소의 주소를 그대로 홈페이지 주소로 사용할 수 있는 기능이다.
    미리 만들어진 테마를 사용할 수도 있다. 이 기능을 사용하면 깃허브 저장소를
    웹 서버처럼 활용할 수 있기 때문에 무료로 깃허브의 저장소를 블로그나 홈페
    이지 공간으로 사용할 수 있다.
    
    개인컴퓨터                     웹서버                    웹 브라우저
    __________   --------------> _________     -----------> ____________
    index.html       push        index.html                    방문자

    GitHub Pages는 정적인(static) 페이지를 제공할 때 사용하기 좋다. 정적인
    페이지란 게시판이나 회원 가입 등이 없고, 내용을 바꾸려면 서버에 올라가 있는
    파일을 수정해야 하는 페이지를 말한다. 그동안의 작업 결과를 보여주는
    포트폴리오 사이트나 기술 정보를 정리해 놓은 블로그 처럼...
    기본적으로 GitHub Pages로 만든 주소에는 woowabros.github.io처럼 .github.io
    라는 이름이 붙는다. 아래 사이트는 국내의 한 기업에서 GitHub Pages를 통해
    운영하는 기술 블로그이다.
    반응형 웹 디자인을 만드는 Bootstrap이라는 오픈 소스의 
    사이트(https://getbootstrap.com/)도 GitHub Pages를 사용해 만든 홈페이지이다.

GitHub Pages를 사용하는 두 가지 방법
    GitHub Pages를 사용하는 방법은 크게 두 가지가 있다. 첫 번째는 HTML과 CSS,
    Javascript(또는 jQuery)등 홈페이지 파일이 있는 경우이다. 이 때는 직접 준비한
    파일을 바로 깃허브 저장소에 올리고 GitHub Pages 기능을 사용해 홈페이지를 
    열면 된다. 두 번째로 깃허브에서 지원하는 지킬 테마(Jekyll theme)를 가져다가
    사용하는 방법이 있다. 홈페이지 파일이 따로 없고 만들 엄두조차 생기지 않는다면
    두 번째 방법을 사용해 보세요.

홈페이지 파일이 있을 때 GitHub Pages 사용하기
    GitHub Pages를 사용하는 두 가지 방법 중 먼저 홈페이지 파일이 준비되었을 경우를
    실습해 보자. 먼저 홈페이지 파일을 저장소에 올리자.
    
    1. 홈페이지 파일 저장소로 올리기
        깃허브에 로그인한 후 계정 옆의 [+]를 누르고 [New repository]를 선택
    2.'Repository name' 필드에 '계정.github.io'라고 입력
        예를 들어 깃허브 계정이 xxx이라면 xxx.github.io라고 입력한다.
        그리고 화면 아래쪽에 있는 'Initialize this repository with:'에서
        Add a README file을 반드시 체크하고 [Create repository]를 누른다.
    3. 파일 목록 위에 있는 [Add file]의 Upload files를 누른다.
    4. 깃허브에 파일을 올리는 두 가지 방법. 
        탐색기에서 올릴 파일을 깃허브 화면 위로 드래그 하는 방법과
        [Choose your files]을 누른 뒤 올릴 파일을 선택하는 방법이다.
        만일 올릴 파일에 하위 폴더가 포함되어 있다면 드래그하는 방법을 사용해야한다.    
    5. css, images, index 등 올릴 파일을 선택한 후 깃허브 화면 위로 드래그
    6. 파일 전송이 모두 끝나면 간단하게 커밋 메시지를 작성한 후 [Commit changes] 클릭
    7. 새로운 저장소에 여러 파일들이 올라왔다. 이 모습만 보면 다른 깃허브 저장소와 
       다를 바가 없다.

    8. 홈페이지 확인하기
        위처럼 파일을 올리기만 해도 홈페이지가 만들어졌을까? 설정을 확인하고 홈페이지에
        직접 접속해 보자.
       
       a. 깃허브 저장소 화면에서 [Settings]를 누른다.
       b. 화면을 아래로 내려서 GitHub Pages 항목을 찾아보자
            사이트가 만들어졌다는 메시지와 함께 홈페이지 주소가 나타난다.
            Your site is published at https://narospace.github.io/
       c. 웹 브라우저에 홈페이지 주소를 입력해서 제대로 열리는지 확인하자.
          주소를 다른 사람에게 알려주고 접속해서 홈페이지 열리는지 확인

홈페이지 파일이 없을 때 GitHub Pages 사용하기
    이제 홈페이지 파일이 따로 없을 경우 미리 만들어 놓은 디자인 중에서 골라 사용할 수
    있는 지킬 테마에 대해 알아보자. 지킬 테마는 GitHub Pages로 블로그나 사이트를 열 때
    쓸 수 있는 디자인 스타일 모음을 말한다. 사용자의 필요에 따라 소스를 수정해서 사용
    할 수 있다.


    지킬 테마 알아보고 선택하기

     지킬 테마 디자인을 선택할 때는 자신이 만들려는 블로그의 목적을 먼저 생각해야 한다.
     블로그를 만드는 목적은 크게 두 가지로 나눈다. 첫 번째는 온라인 이력서 등을 올리는
     경우. 이 경우에는 한 번에 블로그 전체를 살펴볼 수 있도록 디자인 해야 하며 잦은
     수정은 고려하지 않아도 된다.(online-cv 테마)
     두 번째는 포털사이트 블로그처럼 게시물을 자주 작성할 경우이다. 이럴 때는 수정이 편
     해야 하고, 최근 게시물이 앞에 보이도록 해야 한다.

    1. 지킬 테마는 대부분 깃허브 사이트에서 소스를 제공하므로 깃허브 검색 창에서 
       'jekyll'을 검색하면 관련된 소스를 찾을 수 있다.
    2. 검색 결과에 표시된 오픈 소스의 간략한 설명 중에 'blog'나 'theme'이라는 단어가
       있다면 지킬 테마일 가능성이 높다. 검색 결과 중에서 하나를 클릭
    3. 지킬 테마가 담긴 오픈 소스 저장소에는 대부분 테마를 사용한 데모 사이트가 
       링크되어 있다. 
    4. 해당 지킬 테마를 사용했을 때 어떤 형태의 사이트가 열리는지 확인
    5. 여기에서는 jekyll-now로 검색했을 때 가장 먼저 나오는 barryclark 저장소에 있는
       지킬 테마 소스를 사용해 블로그를 만들어 보자. 복잡한 디자인 수정은 여러분의
       HTML과 CSS 실력에 맡기고, 이제부터 블로그를 만드는 방법과 포스트를 작성하는
       방법을 알아보자

       참고| Jekyll Theme 사이트
        깃허브 지킬 테마를 찾아볼 수도 있지만 저장소마다 직접 들어가 살펴보지 않으면
        디자인을 확인하기 어려워 불편하다. 여러 지킬 테마의 디자인을 살펴보기 쉽게
        한눈에 정리해 놓은 Jekyll Themes  (jekyllthemes.org/) 사이트를 이용하면
        디자인을 직접 확인하고 지킬 테마를 고를 수 있다.
        사이트에서 미리보기 화면을 누르면 테마에 대한 자세한 정보도 확인할 수 있고
        [Demo]를 눌러서 전체 디자인을 확인할 수 있으며, [Homepage]를 눌러 깃허브로
        이동하거나 [Download]를 눌러 소스 파일을 내려받을 수 있다.

    블로그 만들고 기본 환경 설정하기
        지킬 테마를 사용하는 방법도 크게 두 가지이다. 지킬 테마 소스를 그대로 포크해서
        사용하는 방법과 소스를 사용자 컴퓨터로 내려받아 사용하는 방법이다.
        지킬 테마 소스를 포크하면 자신의 깃허브 저장소에 소스가 복제되는데, 이 방법은
        테마를 그대로 사용하면서 블로그 포스트만 작성할 경우에 편리하다.
        지킬 테마 소스를 내려받아서 사용하는 방법은 테마 디자인을 바꾸거나 소스를 수정
        하는 것처럼 수정할 파일이 많을 때 적합하다. 여기에서는 테마 소스를 그대로 포크
        해서 사용하는 방법을 알아보자.

        1. 앞에서 들어간 jekyll-now 테마소스 저장소를 자신의 저장소로 포크한다.
        2. 깃허브 계정 안에 jekyll-now 저장소가 복제되었다. 저장소를 GitHub Pages로
           사용하려면 저장소 이름을 '계정.github.io'로 바꿔야 한다.
           복제된 저장소에서 'Settings'를 누른다. 
        3. 'Repository name'항목에 테마 이름이 적혀 있을 것이다. 그 이름을 삭제한 후
           여러분의 깃허브 계정 뒤에 .github.io를 붙여서 저장소 이름을 작성한 후
           [Rename]을 클릭
        4. 지킬 테마를 사용하려면 가장 먼저 환경을 설정해야 한다. 저장소에 있는 파일
           목록 중 '_config.yml'파일 클릭
        5. 파일 내용이 화면에 표시된다. 이 파일을 여러분 환경에 맞게 수정해야 하기 때문에
           연필 아이콘을 클릭
        6. 각 항목마다 주석이 남겨져 있으니 이 내용을 참고하면서 자신과 맞게 수정
           항목들 중에는 name과 description, footer links의 url 부분은 꼭 수정할 것.
           수정한 후에는 간단한 커밋 메시지와 함께 [Commit]을 눌러 커밋 실행
        7. 깃허브 화면이 열린 브라우저 창에 새로운 탭을 열고 블로그 주소를 입력해
           확인하자. _config.yml 에서 수정했던 name과 description 내용이 블로그 위쪽에
           난타나고, footer links에서 지정한 SNS 링크가 블로그 아래쪽에 나타난다.
           단, 변경 내용이 GitHub Pages에 적용되려면 몇 초 ~ 10여 분까지 시간이 걸리므로
           즉시 바뀌지 않더라도 잠시 후에 다시 확인하세요.
        
블로그에 포스트 작성하기
    이제 블로그에 포스트를 작성하는 방법을 알아보자. 지킬 테마에서 블로그 포스트는 _posts
    디렉터리에 저장된다. 깃허브 저장소에서 _posts 디렉터리를 열어 보자. 기본 포스트 파일
    '2014-3-3-Hello-World.md'를 확인할 수 있다. 이렇게 포스트 파일의 이름은 
    '20xx-x-x-파일이름.md'형식을 사용한다. 파일 확장자에서 알 수 있는 것처럼 포스트는
    마크다운 문법을 사용해 작성한다. 이제 포스트를 작성해 보자

    1. 먼저 기본 포스트 파일 '2014-3-3-Hello-World.md'를 클릭
    2. 포스트를 작성할 때도 기본적으로 지켜야 할 형식이 있는데 연필 아이콘 눌러서
       파일이 어떻게 작성되었는지 확인하자.
    3. 포스트 내용에서 ---부터 ---까지는 모든 포스트에 꼭 들어가야 할 내용이다.
       layout:post는 수정하면 안 되고, title 부분은 포스트 제목이므로 포스트를 작성할
       때마다 수정한다. 새 포스트에 사용하기 위해 ---부터 ---까지 선택한 후 복사
    4. 웹 브라우저의 '뒤로'버튼을 두 번 눌러 '_posts'디렉터리 화면으로 돌아오고
       [Add file] Create new file을 클릭
    5. 파일 이름에는 '2020-11-09-first.md'처럼 오늘 날짜와 파일 이름, 그리고 확장자.md
       를 입력. 포스트 내용 맨 위에는 방금 복사했던 내용을 붙여 넣는다. 그리고 title:
       다음의 내용을 수정한다. title 내용은 블로그에 표시되는 포스트의 제목이다.
       그 다음 원하는 내용을 입력한다.
    6. 입력이 끝났으면 화면 아래에 있는 [commit new file]을 눌러 커밋한다.
    7. 새 포스트를 커밋하면 _posts 디렉터리에 방금 만든 포스트 파일이 나타난다.
    8. 블로그에도 포스트가 제대로 표시되는가? 1~2분 정도 기다렸다가 블로그를 확인해 보자.
       'READ MORE'를 누르면 포스트 내용 전체를 볼 수 있다.

포스트에 이미지 삽입하기
    포스트에 글만 작성한다면 너무 밋밋... 작성했던 포스트를 수정해서 이미지를 삽입해 보자.
    포스트에 이미지를 삽입하려면 먼저 이미지 파일을 저장소로 올려야 한다.

    1. 블로그 이미지는 보통 img 디렉터리나 images 디렉터리에 모아서 관리
       jekill-now 테마에서는 images 디렉터리를 사용하고 있다. 저장소의 images 디렉터리를
       누른 후 필요한 파일을 올리고 커밋한다. 
    2. 작성한 포스트를 수정하려면 _posts 디렉터리에서 수정할 포스트 파일을 클릭
    3. 포스트 내용을 수정하기 위해 연필 아이콘 클릭
    4. images 디렉터리에 있는 이미지를 삽입하려면 다음고 같은 형식으로 삽입할 이미지 파일
       을 지정한다.
       ![대체텍스트](/images/파일이름)
    5. 커밋하기 전에 [Preview changes]를 눌러 이미지가 제대로 나나타는지 확인
    6. 커밋 메시지를 입력한 후 [Commit changes]를 눌러 커밋완료
    7. 블로그에서 이미지가 잘 삽입되었는지 확인
       첫 화면에서 포스트 내용에 있는 'READMORE'를 눌러 보자. 즉시 내용이 바뀌지 않는다면
       몇 분 더 있다가 확인
